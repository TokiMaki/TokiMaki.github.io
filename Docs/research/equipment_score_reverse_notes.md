# 장비점수 역산 메모

## 사용 데이터

- 공식 장비점수: `Docs/equipment_score_official_dump.json`의 `equipmentPoint`
- OpenAPI 원재료: `Docs/equipment_score_api_dump.json`
- 병합 표: `Docs/equipment_score_compare.tsv`
- 표본: 카인 서버 36캐릭

버프점수(`buffPoint`)는 장비점수와 별도라 이 메모에서는 제외한다.

## 1차 결론

OpenAPI `status`의 `최종 데미지 증가`가 공식 장비점수와 가장 강하게 맞는다.

```text
상관계수
finalDamageIncrease  0.9688
setPrimevalCount     0.8982
tuneSetPointSum      0.8766
attackAmplification  0.8440
amplificationSum     0.8396
```

명성은 장비점수 산식의 입력값이 아니라 결과적으로 같이 움직이는 별도 지표이므로 역산 후보에서 제외한다.

가장 단순한 근사식은 아래처럼 잡을 수 있다.

```text
장비점수 ~= -22,578 + 17.882 * sqrt(최종 데미지 증가)
```

이 단순식의 오차:

```text
MAE  약 5,610점
RMSE 약 8,651점
```

다만 이 식은 마키로그처럼 직업/스킬 계수가 큰 캐릭터에서 크게 낮게 잡힌다.

## 실전형 근사

현재 표본에서는 장비점수가 아래 형태에 가깝다.

```text
장비점수 ~= sqrt(최종 데미지 증가) * 직업/캐릭터 계수
```

계수는 장비 완성도에 따라 크게 갈린다.

```text
3태초 + 8에픽 완성 캐릭 평균 계수: 13.44
3태초 + 8에픽 미완성 캐릭 평균 계수: 1.34
```

완성 캐릭의 계수 범위는 넓다.

```text
완성 캐릭 계수 범위: 11.08 ~ 19.52
미완성 캐릭 계수 범위: 0.86 ~ 2.35
```

즉 OpenAPI만으로 전 캐릭 공통 공식 하나를 세우면 정확도가 부족하다. 공식 장비점수에는 OpenAPI `status`에 포함된 최종 데미지 계열 외에도 직업/스킬 계수 또는 공식 내부 전투 환산 계수가 들어간 것으로 보는 게 맞다. 명성은 이 보정값으로 쓰지 않는다.

## 구현 판단

사이트에서 OpenAPI만으로 장비점수를 역산하려면 먼저 공식 데미지식을 고정해야 한다. `finalDamageIncrease` 하나로 축약하거나 캐릭터 보정 계수를 두는 방식은 버린다.

```text
1. status에서 데미지 관련 스탯을 가져온다.
2. 데미지 공식으로 기준 데미지 지수를 계산한다.
3. 공식 장비점수와 기준 데미지 지수의 스케일/누락항을 비교한다.
```

## 데미지 공식 구현

공식 커뮤니티 글 `이론적인 장비 옵션 비교` 기준으로 데미지 관련 핵심 축은 다음 7개다.

```text
힘/지능, 물리/마법/독립 공격력, 속성 피해, 공격력 증가, 공격력 증폭, 최종 데미지 증가, 쿨타임 감소
```

현재 구현 파일:

```text
scripts/calculate_equipment_damage_index.py
Docs/equipment_damage_formula.tsv
```

계산식:

```text
selectedStat = max(힘, 지능)
selectedAttackPhysMag = max(물리 공격, 마법 공격)
selectedAttackWithIndependent = max(물리 공격, 마법 공격, 독립 공격)

statRatio = 1 + selectedStat / 250
elementDamageRatio = 1 + 속성피해 / 100
attackIncreaseRatio = 1 + 공격력증가 / 100
attackAmplificationRatio = 1 + 공격력증폭 / 100
finalDamageRatio = 1 + 최종데미지증가 / 100
cooldownDamageRatio = 100 / (100 - 최종쿨타임감소율)
```

기준 1짜리 스킬의 타격 데미지 지수:

```text
damageIndex =
  selectedAttack
* statRatio
* elementDamageRatio
* attackIncreaseRatio
* attackAmplificationRatio
* finalDamageRatio
```

DPS 지수:

```text
damageIndexDps = damageIndex * cooldownDamageRatio
```

공식 장비점수와의 상관:

```text
damageIndexPhysMag                 0.8893
damageIndexPhysMagDps              0.9137
damageIndexWithIndependent         0.9087
damageIndexWithIndependentDps      0.9338
```

주의:

```text
OpenAPI status의 최종 데미지 증가는 이미 정보창 집계값이다.
장비별 최종 데미지 옵션 원문을 다시 곱한 값이 아니다.
```

다음 역산은 `Docs/equipment_damage_formula.tsv`의 `damageIndexWithIndependentDps`를 기준 지수로 두고, 공식 장비점수와의 스케일 차이를 분석한다.
