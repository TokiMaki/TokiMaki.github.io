# 장비점수 역산기 설계 메모

이 문서는 공식 장비점수 역산기 구현 기준이다.

기존 `equipment_score_reverse_notes.md`는 초기 회귀/탐색 메모가 섞여 있으므로, 구현 시에는 이 문서를 우선한다.

## 목표

목표는 공식 장비점수를 복사해서 표시하는 것이 아니라, 특정 투자/교체가 공식 장비점수를 얼마나 올리는지 계산하는 것이다.

핵심 구조:

```text
변경 후 장비점수 ~= 현재 공식 장비점수 * 데미지 전후비 * 기타 고정점 보정
```

데미지 전후비는 공식 장비점수 계산의 파티 플레이 기준 스탯으로 계산한다.

## 공식 기준

공식 가이드 기준 장비점수는 아래 상태를 기준으로 계산된다.

```text
버프 강화 옵션 포함
지역 버프 포함
임의 사양 버퍼의 버프를 받은 파티 플레이 상태
```

버퍼 기준값:

```text
30Lv 버프 스탯: 168,350
30Lv 버프 공격력: 31,215
100Lv 버프 스탯: 297,900
```

현재 검증 결과상, 주스탯 변화 계산에는 30Lv 버프 스탯과 100Lv 버프 스탯을 모두 더해야 한다.

## API 스탯 명칭

Neople API `status` 원문 명칭:

```text
힘
지능
체력
정신력
물리 공격
마법 공격
독립 공격
화속성 강화 / 수속성 강화 / 명속성 강화 / 암속성 강화
공격력 증가
공격력 증폭
최종 데미지 증가
최종 쿨타임 감소율
```

덤프 스크립트 내부 매핑:

```text
힘     -> str
지능   -> int
체력   -> vit
정신력 -> spr
```

## 기본 스탯 테이블

직업별 기본 스탯은 `Docs/jobBaseStat.json`을 사용한다.

예시:

```text
眞 로그: 힘 815
眞 듀얼리스트: 힘 779
```

지역버프 계산에는 캐릭터의 주스탯에 해당하는 기본 스탯을 사용한다.

## 지역버프 공식

현재 검증된 지역버프 공식:

```text
지역버프 주스탯 = floor(3.08 * (마을 주스탯 - 직업 기본 주스탯) + 2886)
```

장비 변경으로 마을 주스탯이 바뀌면 지역버프도 변경 후 마을 주스탯 기준으로 다시 계산한다.
지역버프는 최종 파티 주스탯에 더하기 전에 소수점을 버린다.

마키로그 검증:

```text
마을 힘: 6,679
기본 힘: 815

floor(3.08 * (6679 - 815) + 2886)
= 20,947

실측 지역버프: 20,948
```

주의:

```text
마키로그 실측 지역버프 20,948과 위 floor 계산 20,947은 1 차이가 있다.
지역버프 자체 표시값은 반올림/올림 가능성이 있으나,
장비점수 전후비 계산에서는 floor 처리 후 최종 장비점수도 floor 처리했을 때 실측과 맞는 케이스가 있다.
```

마키듀얼 최신 API 검증:

```text
마을 힘: 6,553
기본 힘: 779

floor(3.08 * (6553 - 779) + 2886)
= 20,669

지역버프 힘 ~= 20,670
```

## 파티 기준 주스탯

장비점수용 파티 주스탯:

```text
파티 주스탯 =
  마을 주스탯
+ 30Lv 버프 스탯 168,350
+ 100Lv 버프 스탯 297,900
+ 지역버프 주스탯
```

주스탯 배율:

```text
주스탯 배율 = 1 + 파티 주스탯 / 250
```

전후비:

```text
주스탯 전후비 =
(1 + 변경후 파티 주스탯 / 250)
/
(1 + 변경전 파티 주스탯 / 250)
```

## 파티 기준 공격력

공식 버퍼 기준의 30Lv 버프 공격력:

```text
30Lv 버프 공격력: 31,215
```

현재 추정:

```text
파티 공격력 = 마을 공격력 + 31,215
```

마키로그 물공 -30 검증:

```text
시뮬 물공: 4,996
파티 물공: 4,996 + 31,215 = 36,211

예상 점수 하락:
150,173 * 30 / 36,211 ~= 124.4

실측 하락: 129
```

이 항목은 근접하지만 추가 검증이 필요하다.

## 속성 강화 / 속성 배율

속성 강화는 API 마을 `status` 값만 쓰면 안 된다.

115제 장비/서약/세트 등으로 던전에서 적용되는 속강이 API 마을 `status`에 빠지는 경우가 있다.

마키로그:

```text
API 암속강: 306
던전/시뮬 암속강: 436
누락 속강: +130
```

마키남슾:

```text
API 속강: 311
장비 누락 속강: +130
계산 기준 속강: 441
```

속성 배율은 인게임 검증상 아래 식을 사용한다.

```text
속성 배율 = 1.05 + 속성강화 * 0.0045
```

참고:

```text
속강 222 증가 시 피해율은 99.9% 증가
=> 속강 1당 0.45%
```

전후비:

```text
속성 전후비 =
(1.05 + 변경후 속강 * 0.0045)
/
(1.05 + 변경전 속강 * 0.0045)
```

## 공격력 증가

공격력 증가는 총합 전후비로 계산한다.

```text
공격력 증가 전후비 =
(1 + 변경후 공격력증가총합 / 100)
/
(1 + 변경전 공격력증가총합 / 100)
```

마키로그 마법석 교체 검증:

```text
기존 마법석: 공격력 증가 3189, 최종 데미지 35.3%
교체 마법석: 공격력 증가 2100, 최종 데미지 30%

총 공격력 증가 차이: -1089
기존 총 공격력 증가: 71104.5

공격력 증가 전후비:
(1 + (71104.5 - 1089) / 100)
/
(1 + 71104.5 / 100)
= 0.984706

실제 장비점수:
150,173 -> 129,910

공격력 증가 하락과 최종 데미지 하락을 곱하면 실제 하락과 거의 일치.
```

## 최종 데미지 증가

OpenAPI `status`의 `최종 데미지 증가`는 이미 정보창 집계값이다.

현재 장비점수 증분 계산에서는 장비별 최종뎀을 다시 분해해서 곱하지 않고, 전후 총합 또는 특정 옵션 교체 전후비를 사용한다.

단일 옵션 교체 예시:

```text
최종 데미지 1% -> 2%

전후비 = 1.02 / 1.01
```

마키로그 목걸이 마부 검증:

```text
현재 점수: 150,173
전후비: 1.02 / 1.01 = 1.00990099
예상 증가: +1,486.86
```

## 스킬 레벨 옵션 환산

칭호의 `1~95Lv 모든 스킬 Lv +1`은 장비점수 계산에서 최종 데미지 증가 8% 상당으로 환산한다.

마키헌터 `프로스트의 전설` 검증:

```text
칭호 장착 장비점수: 96,755
칭호 제거 장비점수: 77,246

전체 전후비:
96,755 / 77,246
= 1.2525567667
```

칭호의 계산 가능한 옵션:

```text
힘 +100
칭호 마부 힘 +25
모든 속성 강화 +10
칭호 마부 수속성 강화 +6
공격력 증폭 +20%
1~95Lv 모든 스킬 Lv +1
```

Neople item detail API에서는 같은 칭호의 `itemBuff.reinforceSkill`에 `1~95Lv +1`이 있고, `itemStatus`에는 `최종 데미지 증가 8%`가 함께 내려온다.

따라서 장비점수 계산에서는 아래처럼 처리한다.

```text
1~95Lv 모든 스킬 Lv +1
= 장비점수상 최종 데미지 증가 8%
```

마키헌터 기준 `힘 +125`, `수속강 +16`, `공격력 증폭 +20%`, `스킬Lv +1 환산 최종 데미지 +8%`를 적용하면:

```text
floor(77,246 * 계산 전후비)
= 96,755
```

### 플래티넘 칭호

플래티넘 칭호의 `1~95Lv 모든 스킬 Lv +1`은 장비점수 계산에서 최종 데미지 증가 9.6% 상당으로 환산한다.

마키로그 `프로스트의 전설 플래티넘[35Lv]` 검증:

```text
칭호 장착 장비점수: 156,400
칭호 제거 장비점수: 123,060

전체 전후비:
156,400 / 123,060
= 1.2709247522
```

Neople item detail API:

```text
itemBuff.reinforceSkill:
  1~95Lv 모든 스킬 Lv +1

itemStatus:
  최종 데미지 증가 9.6%
```

칭호의 계산 가능한 옵션:

```text
힘 +100
칭호 마부 힘 +25
모든 속성 강화 +10
칭호 마부 암속성 강화 +6
공격력 증폭 +20%
1~95Lv 모든 스킬 Lv +1
```

장비점수 계산에서는 아래처럼 처리한다.

```text
플래티넘 칭호 1~95Lv 모든 스킬 Lv +1
= 장비점수상 최종 데미지 증가 9.6%
```

마키로그 기준 `힘 +125`, `암속강 +16`, `공격력 증폭 +20%`, `스킬Lv +1 환산 최종 데미지 +9.6%`를 적용하면:

```text
floor(123,060 * 계산 전후비)
= 156,400
```

일반 칭호 대비 플래티넘 추가분:

```text
1.096 / 1.08
= 1.0148148148

일반 칭호 대비 약 +1.48%
```

### 크리쳐 레벨 태그

크리쳐의 레벨 태그 옵션은 칭호처럼 `itemBuff.reinforceSkill`로 내려오지 않고, Neople item detail API의 `itemStatus`에 최종 데미지 증가 환산값으로 바로 내려온다.

마키로그 `운명을 담는 여신[35Lv]`:

```text
itemStatus:
  힘 +150
  모든 속성 강화 +40
  공격력 증폭 +40%
  버프력 증폭 +10%
  최종 데미지 증가 +1.5%

itemBuff:
  null
```

마키헌터 일반 크리쳐 `운명을 담는 재단사`:

```text
itemStatus:
  힘 +150
  모든 속성 강화 +40
  공격력 증폭 +40%
  버프력 증폭 +10%

itemBuff:
  null
```

따라서 현재 검증 규칙:

```text
크리쳐 [35Lv] 옵션
= 장비점수상 최종 데미지 증가 1.5%
```

## 쿨타임 감소

쿨감은 일반 DPS식 그대로 쓰면 장비점수와 맞지 않는다.

검증 케이스:

```text
마키여긒
창조의 안개 결정 - 반지:
  최종 데미지 8%
  스킬 쿨타임 감소 20%

정화 : 선명한 광휘 결정:
  최종 데미지 20%

실제 장비점수:
51,239 -> 51,670
증가량 +431
```

`쿨감 20%`를 장비점수상 `최종 데미지 11%` 상당으로 보면:

```text
기존 환산 = 1.19
교체 환산 = 1.20

전후비 = 1.20 / 1.19 = 1.00840336
예상 증가 = +430.58
실제 증가 = +431
```

현재 임시 규칙:

```text
쿨감 20% ~= 장비점수상 최종 데미지 11%
```

주의:

```text
이 규칙은 검증 케이스 1개 기반이다.
쿨감 10/15/20/30 등 추가 표본으로 별도 곡선을 만들어야 한다.
```

## 아바타 / 엠블렘

아바타 엠블렘은 장비점수 계산과 실제 데미지 계산을 분리한다.

장비점수 계산에서는 노란빛/녹색빛 엠블렘 슬롯에 실제 주스탯 엠블렘이 박혀 있지 않아도, 해당 슬롯에 가능한 자기 주스탯 엠블렘이 박힌 것처럼 평가되는 것으로 본다.

현재 규칙:

```text
찬란한 노란빛/녹색빛 엠블렘: 개당 주스탯 +15
화려한 노란빛/녹색빛 엠블렘: 개당 주스탯 +10
```

주의:

```text
이 규칙은 장비점수 계산 전용이다.
실제 데미지 계산에서는 실제로 주스탯 엠블렘이 박혀 있지 않으면 주스탯 증가로 계산하지 않는다.
```

검증 케이스:

```text
마키로그
두 부위 제거 후 힘 -20 상태: 156,336
추가로 얼굴/목가슴 쪽 비교 후: 156,297
차이: -39
```

해석:

```text
얼굴/목가슴에 실제 공격속도 엠블렘이 박혀 있어도,
장비점수 계산은 해당 슬롯에 가능한 힘 엠블렘이 박힌 것으로 계산하는 듯하다.
```

현재 파티 스탯 공식으로 힘 -30을 계산:

```text
예상 하락 ~= -38.73
실측 하락 = -39
```

임시 규칙:

```text
얼굴/목가슴 공격속도 엠블렘 2개는 장비점수상 힘 엠블렘 2개, 즉 힘 30 상당으로 평가된다.
```

향후 구현 시 엠블렘은 `equipmentScoreVirtualStat`과 `realDamageStat`을 분리해서 계산한다.


### 레어아바타 상의/하의 장비점수 환산

레어아바타 상의/하의는 실제 옵션에 주스탯이 보이지 않아도, 장비점수 계산에서는 부위별 주스탯 +30이 포함된 것으로 보는 쪽이 표본 간 일관성이 높다.

검증 표본:

```text
마키듀얼
하의 제거: 139,422 -> 137,173
상의 제거: 139,422 -> 134,992

마키로그
하의 제거: 166,642 -> 163,955
상의 제거: 166,642 -> 161,348
```

주스탯 +30을 장비점수용 파티 스탯 전후비로 제거한 순수 상승률:

```text
하의 레어아바타: 약 +1.6140%
상의 레어아바타: 약 +3.2558%

두 표본 간 차이:
하의: 약 0.0008%p
상의: 약 0.0007%p
```

임시 규칙:

```text
레어아바타 하의 = 장비점수상 주스탯 +30 + 순수 데미지 약 1.614%
레어아바타 상의 = 장비점수상 주스탯 +30 + 순수 데미지 약 3.256%
```

### 마키로그 얼굴/목가슴 엠블렘 포함 검증

얼굴/목가슴 2부위의 노란빛/녹색빛 엠블렘 4개를 모두 찬란한 주스탯 엠블렘으로 보면 장비점수용 가상 힘은 +60이다.


## 구현 순서 제안

1. `jobBaseStat.json`을 로드한다.
2. API `status`에서 마을 주스탯, 공격력, 속강, 공격력 증가, 공격력 증폭, 최종 데미지 증가, 쿨감 값을 읽는다.
3. 장비/세트/서약에서 API 마을 `status`에 누락된 던전 적용 속강을 더한다.
4. 지역버프를 계산한다.
5. 파티 주스탯/공격력을 만든다.
6. 변경 전후 옵션을 데미지 전후비로 바꾼다.
7. 현재 공식 장비점수에 전후비를 곱한 뒤 최종 장비점수는 floor 처리한다.
8. 쿨감/아바타/엠블렘/세트 고정점은 별도 검증표를 기반으로 추가한다.

## 서약의 가호 포인트 보정

서약의 가호 환산값은 `Docs/OATH_VALUE_REVERSE_TABLE.xlsx`의 등급별 시작값을 기준으로 계산한다.

같은 등급 구간 안에서 서약포인트가 25 올라갈 때마다 아래 값을 더한다.

```text
최종 데미지 증가 +0.5%
버프력 +70
명성 +35
```

계산식:

```text
구간 내 단계 수 = floor((현재 서약포인트 - 해당 등급 구간 시작 포인트) / 25)

최종데미지증가 = 등급 시작 최종데미지증가 + 구간 내 단계 수 * 0.5
버프력 = 등급 시작 버프력 + 구간 내 단계 수 * 70
명성 = 등급 시작 명성 + 구간 내 단계 수 * 35
```

구간이 바뀌면 이전 구간 보정은 누적하지 않고 새 등급 시작값에서 다시 계산한다.

예:

```text
레전더리 시작 최종데미지 39
현재 서약포인트 1700
=> 최종데미지 41

2100점이 다음 구간 시작점이면
=> 최종데미지 50, 버프력 8210, 명성 4105를 그대로 사용
```
