# DunPilot 공개 콘텐츠 기획

## 상태

- 공개 콘텐츠의 기본 방향을 정리한 기획 문서다.
- 콘텐츠 구현보다 `src/dnfHellTool/enchantView.js`의 구조 정리를 먼저 진행한다.
- 이 문서는 구현 명세가 아니다. 실제 저장 필드와 집계 가능 여부는 구조 정리 후 코드와 응답 payload를 확인해 확정한다.

## 목적

던파일럿의 기존 강점인 캐릭터 분석과 골드 효율 계산을 활용해, 캐릭터를 검색하지 않은 방문자도 볼 수 있는 공개 콘텐츠를 만든다.

공개 콘텐츠는 다음 흐름으로 기존 분석 기능과 연결한다.

1. 랭킹과 점수 구간별 통계로 다른 캐릭터의 성장과 세팅 분포를 본다.
2. 캐릭터를 검색하면 동일 점수 구간에서 자신의 위치를 확인한다.
3. 기존 스펙업 추천에서 자신의 다음 투자 순서를 확인한다.

단순 시세표, 딜러·버퍼 전투력 랭킹, 정적 스펙업 티어표, 공식 이벤트 내용 복사, 설명형 SEO 문서 양산은 기본 방향에서 제외한다.

## 데이터 범위

- 전체 던파 캐릭터가 아니라 **던파일럿에서 갱신된 캐릭터**만 집계한다.
- 캐릭터별 최신 스냅샷 하나를 통계 표본으로 사용해 중복 집계를 막는다.
- 최초 조회는 비교 기준 스냅샷만 저장하고, 이후 갱신에서 이전 상태와 현재 상태의 차이를 계산한다.
- 딜러의 구간과 성장량은 던파가 제공하는 공식 **장비점수**를 기준으로 한다.
- 버퍼의 구간과 성장량은 던파가 제공하는 공식 **버프점수**를 기준으로 한다.
- 딜러와 버퍼의 통계와 랭킹은 분리한다.
- 서버와 캐릭터명은 공개 조회 정보로 표시할 수 있다. 다만 누가 조회했는지는 알 수 있으므로 해당 캐릭터의 소유자나 던파일럿 이용자로 표현하지 않는다.

## 콘텐츠 묶음

### 1. 세팅 추정 가치 랭킹

가격 근거가 확인되는 마법부여, 칭호, 크리쳐, 오라, 아바타 등 현재 세팅의 교체가치를 합산한다.

- 실제로 소비한 골드가 아니다.
- 캐릭터 또는 계정의 거래가치가 아니다.
- 화면에는 `가격 산정 가능 항목의 현재가 기준 추정 가치`라고 표시한다.
- 이벤트 지급품, 거래 불가능 아이템, 가격 미확인 항목은 근거 없이 환산하지 않는다.
- 확률형 강화·업그레이드의 기댓값을 실제 지출액처럼 합산하지 않는다.

### 2. 이번 주 성장왕

이전 갱신 대비 공식 점수 상승량을 집계한다.

- 딜러: 장비점수 상승량
- 버퍼: 버프점수 상승량
- 실제 변경 시점은 두 조회 사이의 어느 시점인지 알 수 없으므로 `이번 주에 성장한 캐릭터`가 아니라 `이번 주 갱신에서 성장이 확인된 캐릭터`로 설명한다.
- 비교 기간이 지나치게 긴 캐릭터를 같은 주간 집계에 포함할지 별도 기준이 필요하다.

### 3. 이번 주 큰손

이전 갱신에서 새로 확인된 세팅 변경의 현재가 기준 추정 가치를 합산한다.

- `쓴 골드`가 아니라 `갱신에서 확인된 세팅 가치 증가`로 표현한다.
- 변경 항목, 가격 산정 항목, 가격 미확인 항목을 함께 보여준다.

### 4. 이번 주 가성비왕

가격 산정 가능한 세팅 가치 증가 대비 공식 점수 상승량을 비교한다.

- 딜러와 버퍼는 별도 산식과 별도 랭킹을 사용한다.
- 가격 미확인 항목이 많은 캐릭터가 유리해지지 않도록 집계 가능 범위와 제외 기준이 필요하다.
- 점수 상승과 비용의 관계를 충분히 설명할 수 있을 때만 공개한다.

### 5. 공식 점수 구간별 세팅 현황

공식 점수가 비슷한 캐릭터들이 실제로 어느 정도의 세팅을 갖추고 있는지 분포로 보여준다.

딜러는 장비점수 구간, 버퍼는 버프점수 구간을 사용한다.

표시 후보:

- 마법부여 보급형·준종결·종결 사용률과 부위별 분포
- 증폭 수치의 중앙값과 구간별 비율
- 장비 조율과 서약 조율 수준
- 칭호, 크리쳐, 오라, 아바타 등급 또는 보유 분포
- 버퍼의 버프강화와 플래티넘 관련 분포
- 표본 수, 집계 기간, 마지막 갱신 시각

평균 세팅을 권장 세팅으로 표현하지 않는다. 평균 하나보다는 중앙값, 상·하위 구간, 단계별 비율을 우선한다.

캐릭터 검색 후에는 동일 점수 구간과 비교한 자신의 위치를 보여주고 기존 스펙업 추천으로 연결할 수 있다.

### 6. 최근 많이 확인된 스펙업 변화

최근 갱신 diff에서 자주 발견된 변경 영역을 집계한다.

예시 영역은 마법부여, 증폭, 장비 조율, 서약 조율, 칭호, 크리쳐, 오라, 아바타다. 실제 지원 범위는 안정적으로 비교 가능한 snapshot 필드를 확인한 뒤 정한다.

## 공통 표시 원칙

- `전체 던파 유저`가 아니라 `최근 던파일럿에서 갱신된 캐릭터 기준`이라고 표시한다.
- 실제 지출액, 실제 변경일, 캐릭터 소유자의 던파일럿 이용 여부를 추정하지 않는다.
- 모든 랭킹과 통계에 표본 수, 집계 기간, 가격 기준 시각을 표시한다.
- 표본이 적은 점수 구간이나 직업 구간은 공개하지 않거나 상위 구간으로 합친다.
- 가격을 확인할 수 없는 항목을 임의의 값으로 채우지 않는다.
- 공개 페이지의 캐릭터 상세에는 점수 변화, 변경 항목, 추정 가치의 산정 범위를 함께 보여준다.
- 공개 콘텐츠 추가가 애드센스 승인을 보장한다고 전제하지 않는다.

## 예상 데이터 흐름

```text
캐릭터 갱신
→ 비교용 스냅샷 정규화
→ 이전 스냅샷과 현재 스냅샷 비교
→ 공식 장비점수·버프점수 변화 계산
→ 가격 산정 가능한 세팅 변경만 추정 가치 계산
→ 캐릭터별 최신 상태와 갱신 이벤트 저장
→ 랭킹과 점수 구간별 통계 집계
→ 공개 콘텐츠에서 캐릭터 분석으로 연결
```

## 구현 전에 결정할 사항

1. 장비점수와 버프점수의 구간 폭
2. 주간 랭킹에 허용할 이전 스냅샷의 최대 경과 기간
3. 비교용 snapshot에 저장할 최소 필드와 보존 기간
4. 가격 산정이 가능한 스펙업 범위와 가격 기준 시각
5. 점수 구간과 랭킹을 공개할 최소 표본 수
6. 동일 캐릭터의 중복 갱신과 비정상적인 반복 갱신 처리
7. 캐릭터 상세 공개 범위와 숨김 요청 처리 방식

## 선행 작업

콘텐츠용 저장·집계 로직을 추가하기 전에 1만 줄을 넘은 `src/dnfHellTool/enchantView.js`의 현재 책임과 호출 구조를 먼저 확인한다. 한 번에 재작성하지 않고, 기존 동작을 보존하면서 한 영역씩 분리한다.
